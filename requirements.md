# Требования к проекту: Оценщик вероятной оценки студента

## Статус: в разработке

---

## 1. Общее описание

Веб-приложение для университета с двумя независимыми страницами:

- **Страница 1 — Прогноз оценки**: студент вводит параметры и получает прогнозируемую оценку по предмету
- **Страница 2 — Журнал семестров**: студент ведёт личный журнал прошлых семестров с фактическими оценками

| Параметр | Значение |
|---|---|
| Бэкенд | Go, REST API |
| Фронтенд | Vue.js |
| Язык интерфейса | Русский |

---

## 2. Входные параметры (предварительно)

> ⚠️ Финальный список параметров ожидается от аналитика.

Используются на обеих страницах.

| Параметр | Описание |
|---|---|
| Время на лекциях | Количество времени, проведённого на лекциях |
| Внимательность на лекциях | Степень внимательности студента во время лекций |
| Самостоятельная подготовка | Количество времени, уделённого самостоятельному изучению |
| Сторонние источники | Изучение дополнительных материалов по предмету |
| Количество сна | Среднее количество сна студента |

---

## 3. Выходные данные

- **Формат вывода**: одна итоговая оценка
- **Шкала**: пятибалльная (2, 3, 4, 5)

---

## 4. Авторизация

- Доступ к системе только для авторизованных пользователей
- Неавторизованные запросы отклоняются
- Механизм: логин/пароль
- Ролей нет — все пользователи равнозначны

---

## 5. Страница 1 — Прогноз оценки

- Пользователь вводит параметры и получает прогнозируемую оценку
- Каждый расчёт автоматически сохраняется в историю
- Пользователь может просматривать свою историю расчётов
- Пользователь может очистить всю историю одним действием (кнопка "очистить всё")

---

## 6. Страница 2 — Журнал прошлых семестров

- Пользователь вручную добавляет записи о прошедших семестрах
- Страница независима от страницы 1
- Редактирование записей не предусмотрено
- Каждая запись содержит:
  - Название предмета
  - Фактически полученная оценка (2–5)
  - Входные параметры (те же что на странице 1)
- Доступные действия: добавить запись, просмотреть список, удалить запись

---

## 7. Технические требования к API

- **Формат**: JSON
- **Версионирование**: префикс `/api/v1/`
- **CORS**: разрешён origin `http://localhost:8080`
- **Валидация входных данных**: на стороне API
- **Пагинация**: отсутствует, все записи возвращаются одним списком

---

## 8. База данных

- **СУБД**: PostgreSQL (одна БД)
- Таблицы:
  - `users` — пользователи (логин/пароль)
  - `predictions` — история расчётов (страница 1)
  - `semesters` — журнал прошлых семестров (страница 2)

---

## 9. Инфраструктура и деплой

- Запуск через **Docker** (docker-compose)
- Цель: локальный запуск для сдачи проекта
- Срок сдачи: **15 мая 2026**

## 10. Эндпоинты API

### Авторизация

| Метод | Путь | Описание |
|---|---|---|
| POST | `/api/v1/auth/register` | Регистрация |
| POST | `/api/v1/auth/login` | Вход |

### Страница 1 — Прогноз

| Метод | Путь | Описание |
|---|---|---|
| POST | `/api/v1/estimate` | Расчёт вероятной оценки |
| GET | `/api/v1/predictions` | История расчётов |
| DELETE | `/api/v1/predictions` | Очистить всю историю |

### Страница 2 — Журнал семестров

| Метод | Путь | Описание |
|---|---|---|
| GET | `/api/v1/semesters` | Список записей |
| POST | `/api/v1/semesters` | Добавить запись |
| DELETE | `/api/v1/semesters/:id` | Удалить одну запись |

### 11.2 Вопросы к аналитику

- [ ] Финальный список входных параметров и их типы (числовые, категориальные, булевы?)
- [ ] Диапазоны допустимых значений для каждого параметра

- [ ] **Алгоритм расчёта оценки** _(критический вопрос)_

  Необходимо понять, каким образом входные параметры (время на лекциях, внимательность, сон и т.д.) преобразуются в итоговую оценку по пятибалльной шкале. Конкретно нужно получить ответ на следующее:
  - Это **статическая формула** (например, взвешенная сумма параметров с порогами) или **ML-модель** (обученная на реальных данных студентов)?
  - Если формула — каковы **веса каждого параметра** и **пороговые значения** для каждой оценки (2/3/4/5)?
  - Если ML-модель — на каком датасете обучена, какой алгоритм используется, в каком формате предоставляется модель (pickle, ONNX, встроенная в Go-сервис)?
  - Все ли параметры **равнозначны** или одни влияют на оценку сильнее других?
  - Формула **универсальная** для любого предмета или зависит от дисциплины?
